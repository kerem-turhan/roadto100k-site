## Outcome (same day)

All four phases shipped on 2026-07-22. Test suite 122 → 288; `npm audit --omit=dev` clean; CI
runs `typecheck → lint → test → leaks --git → build → leaks --dist`, plus a `pre-push` hook so
a leak is caught before the commit is public rather than after.

Every mutation that passed during this audit now fails: a placeholder proof URL (2 tests), an
unpinned numeric claim (4), an impossible calendar date (1), a config/ledger date divergence
(1), an absolute URL pasted back into index.html (3 tests + the build), a forbidden term in a
source file, a comment or generated output (`leaks`, exit 1, redacted), and 15 distinct "the
guard could not look" conditions.

Two findings stay open, both cosmetic, both recorded in ANLATIM §5: the `№ ← → ↗` glyphs still
fall back to a system font, and the header brand shares a destination with a footer link.

Two regressions were introduced and fixed inside the session, which is worth recording because
both were the same shape as the bugs being fixed:

1. Dropping `apply: 'build'` from the static-pages plugin — so dev could fill the `%SITE_URL%`
   token — also let vitest reach `closeBundle`, and vitest's default `build.outDir` is a
   placeholder directory. Every `npm run test` quietly wrote a copy of the site into the repo
   root. The plugin is now split, and the writing half refuses any output directory but `dist/`.
2. The leak guard's own denylist was committed. Salted hashes felt safe; they were not. The
   file also carried the salt, a known-plaintext canary confirming the construction, each
   term's exact length and a note naming its category — enough to brute-force a term in
   seconds and enough to confirm any single guess. The terms now come from a CI secret or a
   gitignored local file, missing means exit 1, the stored shape is `{len, hash}` only, the
   old shape is refused rather than ignored, the canary moved to its own salt in code, the
   file was stripped from all 24 commits with `git filter-repo`, and the salt was rotated so
   the hashes that were briefly public are inert.

The second one is the sharper lesson of the day: the guard built to enforce a secrecy rule was
itself the leak. "It's hashed" is a claim like any other — it needed a test, and the test is
whether someone can recover the input. Someone could, in seconds.
